## Timeline 系统

- 数据存储在 `timelines/` 目录
- 文件名格式：`YYYY-MM-DD.md`，无日期待办用 `0000-00-00.md`
- 每个文件有三个 H2 章节：`## Events`、`## Todos`、`## Notes`
- 数据格式和完整性约束见 `timeline` skill（按需加载）

## 操作流程

### 创建待办

1. 解析日期和时间（见时间解析规则）
2. 确定目标文件（不存在则用模板创建）
3. 在 `## Todos` 章节添加条目
4. 如果待办是**今天**的且有时间，创建对应的 cron job
5. 显示文件名和更新内容

### 创建事件

1. 事件时间取**当前时间**（除非用户指定过去时间）
2. 在目标文件的 `## Events` 章节添加条目
3. 主句精简，细节折叠到缩进详情
4. 如果是未来承诺的事件，询问用户是否创建对应待办
5. 显示文件名和更新内容

### 完成待办

1. 创建一个 Event 记录完成的动作
2. 在所有文件中搜索匹配的 Todo（包括 `0000-00-00.md`）
3. 如果恰好匹配一条：标记为 `[x]`
4. 如果匹配多条：询问用户选哪条
5. 如果没有匹配：只创建 Event，告知用户
6. 检查该时间的 cron 是否还有其他待办，无则删除

### 放弃待办

1. 给待办描述加删除线：`- [ ] ~~描述~~`
2. 同"完成待办"的 cron 清理流程

### 移动待办

1. 解析目标日期/时间
2. 从原文件删除待办
3. 在目标文件添加待办
4. Cron 处理：删旧 cron（若无其他待办），建新 cron（若新时间是今天）

### 编辑条目

1. 搜索匹配的条目
2. 修改时间或描述
3. 如果修改了时间：同"移动待办"的 cron 处理
4. 如果只修改描述（时间不变）：不操作 cron

### 删除待办

1. 搜索匹配的 Todo
2. 如果匹配多条：询问用户选哪条
3. 删除匹配的 Todo 行
4. Cron 清理：检查该时间是否还有其他待办，无则删除

## 时间解析规则

| 用户说法 | 解释 |
|---------|------|
| 「三点」 | 15:00（默认下午） |
| 「凌晨三点」 | 03:00 |
| 「早上九点」 | 09:00 |
| 「今晚八点」 | 20:00 今天 |
| 「明天」 | 明天 |
| 「后天」 | 后天 |
| 「下周一」「这周五」 | 具体星期 |
| 「6月15日」 | 具体日期 |

**模糊情况**：务必询问确认（如「下周开会」→「请问是下周几？」）

## Cron 配置

根据 ADR-0001，cron 直接触发 timeline skill。

### 定时提醒

- **命名格式**：`timeline-{YYYYMMDD}-{HHMM}`
- **触发时机**：创建今天有时间的待办时
- **触发方式**：skill 响应"有什么到期的待办"
- **清理时机**：完成/取消/删除待办后，检查该时间是否还有其他待办

### 每小时扫描

- **命名**：`timeline-hourly`
- **触发频率**：每小时
- **触发方式**：skill 响应"过期的待办有什么"

## 用户偏好

### 深夜时段确认

当前时间在 00:00-05:00 时：
- 用户说「今天」可能指昨天（还没睡觉）
- 用户说「明天」可能指今天
- **务必确认**用户实际指哪天

### 回复格式

操作完成后显示：
```
已在 YYYY-MM-DD.md 创建待办：
- [ ] HH:MM 描述
    - 详情（如有）
```

### 实时日记模式

用户快速连续报告事件时，回复保持极简（表情 + 内容），不每次展示完整文件。

## 辅助脚本

以下脚本用于数据校验和查询（位于 timeline skill 的 scripts 目录）：

- `doctor.py`：验证数据格式，`--fix` 自动修复
- `todo_list.py`：列出所有未完成待办
- `todo_by_time.py`：按时间提取待办
- `todo_overdue.py`：提取过期和无时间待办
